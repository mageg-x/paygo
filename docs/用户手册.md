# PayGo 支付系统用户手册

## 系统简介

PayGo 是一个支付网关平台，支持支付宝、微信支付等多种支付渠道。系统采用 **商户-渠道-插件** 三层架构。

## 角色说明

系统有两个角色：

| 角色 | 入口 | 说明 |
|------|------|------|
| 管理员 | `/admin` | 管理系统配置、通道、插件、商户 |
| 商户 | `/user` | 接入支付、查看订单、申请结算 |

---

## 商户使用指南

### 1. 商户注册与登录

1. 访问商户后台 `/user/login`
2. 输入商户 ID（由管理员分配）和密码登录
3. 支持密钥登录（更安全）

### 2. 商户资料配置

登录后进入 **资料管理**：

- **基本信息**：修改用户名、邮箱、手机、QQ
- **实名认证**：提交身份证信息（待审核）
- **API信息**：查看商户ID和API密钥，用于对接

### 3. 支付接入

商户通过 API 对接系统，示例：

```bash
# 创建支付订单
POST /api/pay/create
{
  "uid": 1001,          # 商户ID
  "key": "xxx",         # API密钥
  "trade_no": "order_123",  # 商户订单号
  "money": 100.00,      # 金额
  "type": 1,            # 支付类型 (1=支付宝, 2=微信)
  "notify_url": "https://yourapp.com/notify"
}
```

### 4. 订单管理

- 查看所有交易订单
- 按状态筛选（待支付/已支付/已退款）
- 查看订单详情（交易号、金额、时间）

### 5. 结算管理

商户账户余额支持提现：

1. 进入 **结算管理**
2. 点击 **申请结算**
3. 填写收款账户信息（支付宝/微信/银行卡）
4. 填写提现金额
5. 提交申请，等待管理员审核

### 6. 资金记录

查看所有资金变动明细：
- 收入/支出
- 充值/提现
- 手续费扣除

## 管理员指南

### 1. 系统设置

进入 **系统设置** 页面，配置平台参数：

| 标签页 | 配置项 | 说明 |
|--------|--------|------|
| 网站信息 | 网站名称、标题、回调地址、API地址 | 平台基本信息 |
| 支付设置 | 测试支付开关 | 是否允许测试金额 |
| 结算设置 | 结算周期、最低结算金额、结算费率 | 商户提现规则 |
| 转账设置 | 代付开关、手续费 | 商户代付功能 |
| OAuth设置 | 第三方登录配置 | 用户第三方登录 |
| 通知设置 | 邮件/短信通知 | 订单/结算通知 |
| 账号设置 | 管理员账号 | 修改管理员密码 |

### 2. 通道管理

**通道** = 支付渠道（如支付宝、微信支付）

进入 **通道管理**：

| 字段 | 说明 |
|------|------|
| 通道名称 | 如"支付宝当面付" |
| 插件 | 绑定的支付插件 |
| 费率 | 商户使用该通道的手续费 |
| 状态 | 启用/禁用 |

**添加通道步骤**：
1. 点击 **添加通道**
2. 选择插件类型（alipay/wxpay）
3. 填写通道名称和费率
4. 配置通道参数（AppID、密钥等）
5. 设置限额（单笔最小/最大、每日限额）
6. 保存

### 3. 插件管理

**插件** = 内置支付能力（代码层面实现）

系统内置插件：
- `alipay` - 支付宝
- `wxpay` - 微信支付

插件状态由管理员控制：
- **刷新**：重新读取插件信息
- **启用/禁用**：控制插件是否可用
- **配置**：为插件设置全局参数

### 4. 商户管理

进入 **商户管理**：

| 字段 | 说明 |
|------|------|
| UID | 商户ID（唯一） |
| 商户名称 | 显示名称 |
| 分组 | 所属用户组（决定费率） |
| 余额 | 账户余额 |
| 支付权限 | 是否允许收款 |
| 结算权限 | 是否允许提现 |
| 状态 | 正常/禁用/待审核 |

**商户操作**：
- **编辑**：修改商户信息、费率、权限
- **重置密钥**：生成新的API密钥
- **启用/禁用**：控制商户登录和交易
- **代付权限**：是否允许代付功能

### 5. 订单管理

查看所有商户的交易订单：

| 字段 | 说明 |
|------|------|
| 订单号 | 系统订单号 |
| 商户订单号 | 商户自定义订单号 |
| 商户 | 所属商户 |
| 金额 | 交易金额 |
| 状态 | 待支付/已支付/已退款/已关闭 |
| 时间 | 创建/支付时间 |

可按商户、状态、订单号搜索筛选。

### 6. 结算管理

商户提现申请处理：

| 状态 | 说明 |
|------|------|
| 待审核 | 商户发起，等待处理 |
| 已通过 | 审核通过，已打款 |
| 已拒绝 | 审核拒绝 |

**处理流程**：
1. 查看结算申请详情（金额、收款账户）
2. 选择 **通过** 或 **拒绝**
3. 拒绝需填写原因

### 7. 转账管理（代付）

平台代付给商户或用户：

- 查看转账记录
- 处理待转账申请
- 批量转账

## 常见问题

### Q: 如何添加新的支付渠道？

1. **开发插件**：在 `server/src/plugin/channels/` 下实现新插件
2. **导入插件**：在 `plugins.go` 中 import 插件包
3. **添加通道**：在管理后台添加通道，选择新插件
4. **配置参数**：为通道设置 AppID、密钥等

### Q: 商户如何申请结算？

1. 商户后台 → 结算管理 → 申请结算
2. 填写收款账户和金额
3. 管理员审核通过后打款

### Q: 不同商户可以有不同费率吗？

可以。通过 **用户组** 设置不同费率：
- 创建用户组，设置结算费率
- 编辑商户时分配到对应组
- 商户使用组内配置的费率结算

### Q: 如何控制商户的支付权限？

管理员可单独控制每个商户：
- **支付权限**：是否允许收款
- **结算权限**：是否允许提现
- **代付权限**：是否允许代付
- **状态**：正常/禁用/待审核

### Q: 回调地址是什么？

回调地址是商户服务器接收支付通知的 URL：
1. 商户发起支付时传入 `notify_url`
2. 支付成功后系统 POST 通知到该地址
3. 商户服务器处理逻辑（发货等）

## API 对接示例

### 1. 支付下单

```bash
POST /api/pay/create
Content-Type: application/json

{
  "uid": "1001",
  "key": "your_api_key",
  "trade_no": "20240301001",
  "money": 100.00,
  "type": 1,
  "notify_url": "https://yourapp.com/pay/notify"
}
```

### 2. 支付回调

系统 POST 到商户 `notify_url`：

```json
{
  "code": 0,
  "trade_no": "20240301001",
  "status": 1,
  "money": 100.00
}
```

### 3. 订单查询

```bash
GET /api/pay/query?trade_no=20240301001&uid=1001&key=your_api_key
```
