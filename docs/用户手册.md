## 1.2 访问地址

| 角色 | 地址 | 默认账号 |
|------|------|----------|
| 管理后台 | http://localhost:8080/admin | admin / 12345678 |
| 商户后台 | http://localhost:8080/user/login | 商户注册的账号 |

## 1.3 首次配置流程

```
第一步：修改管理员密码
   └─ 系统设置 → 账户设置 → 修改密码

第二步：配置支付插件
   └─ 插件管理 → 刷新插件 → 配置插件参数（AppID、密钥等）

第三步：创建支付通道
   └─ 通道管理 → 添加通道 → 选择插件、设置费率

第四步：创建商户
   └─ 商户管理 → 添加商户 → 填写商户信息

第五步：配置用户组（可选）
   └─ 用户组 → 添加/编辑 → 设置费率规则
```

# 第二章：管理后台

## 2.1 首页

**功能**：系统数据概览和快捷操作入口

### 统计卡片说明

| 卡片 | 含义 |
|------|------|
| 今日订单数 | 今日创建的所有订单数量（含已支付+待支付+已关闭） |
| 今日交易额 | 今日已支付订单的总金额 |
| 待处理订单 | 状态为"待支付"的订单数量 |
| 商户总数 | 已注册的商户总数量 |
| 昨日交易额 | 昨天的已支付订单总金额 |

### 快捷操作

点击卡片可快速跳转到对应管理页面：
- 商户管理：跳转至商户列表
- 订单管理：跳转至订单列表
- 结算管理：跳转至结算列表
- 系统设置：跳转至系统设置页面

### 建议

首次配置完成后，每天上班先看首页，了解昨日交易情况。有问题看日志（操作日志菜单）排查。

## 2.3 用户组

**功能**：定义不同商户群体的费率规则

### 2.3.1 为什么需要用户组？

假设你有两类商户：
- 优质商户：交易稳定，希望低费率
- 普通商户：正常费率

可以创建两个用户组，分别设置不同费率：
- 优质商户组：结算费率0.3%
- 普通商户组：结算费率0.5%

商户加入某个组后，所有交易按该组费率结算。

### 2.3.2 用户组列表

页面路径：管理后台 → 用户组

```
┌──────────────────────────────────────────────────────────────────────┐
│ 用户组管理                            [添加用户组]                     │
├──────────────────────────────────────────────────────────────────────┤
│ 名称      │ 描述         │ 结算费率 │ 结算周期 │ 自动结算 │ 状态 │ 操作 │
│ 默认组    │ 所有新商户    │ 0.5%    │ 每日    │   √     │ 启用 │[编辑]│
│ 优质商户  │ 低费率商户    │ 0.3%    │ 每日    │   √     │ 启用 │[编辑]│
│ vip商户  │ 最高优惠      │ 0.2%    │ 实时    │   √     │ 启用 │[编辑]│
└──────────────────────────────────────────────────────────────────────┘
```

### 2.3.3 添加/编辑用户组

| 字段 | 说明 | 必填 | 示例 |
|------|------|------|------|
| 用户组名称 | 显示名称 | 是 | 优质商户 |
| 描述 | 说明文字 | 否 | 交易量大的优质商户 |
| 结算费率(%) | 每笔交易收取的比例 | 是 | 0.5 |
| 最低结算金额 | 单次提现最低金额 | 是 | 30 |
| 结算周期 | 每日结算/实时结算 | 是 | 每日结算 |
| 自动结算 | 是否自动打款 | 否 | 开启 |
| 代付费率(%) | 代付手续费率 | 否 | 0 |
| 转账费率(%) | 转账手续费率 | 否 | 0 |
| 结算风控 | 是否启用风控检测 | 否 | 开启 |
| 状态 | 启用/禁用 | 否 | 启用 |

### 2.3.4 配置案例

**案例1：创建标准商户组**

```
用户组名称：标准商户
描述：标准费率商户
结算费率：0.5%
最低结算金额：30
结算周期：每日结算
自动结算：关闭（需要管理员审核）
代付费率：0
转账费率：0
结算风控：开启
状态：启用
```

**案例2：创建VIP商户组**

```
用户组名称：VIP商户
描述：交易量大，申请审批通过的高质量商户
结算费率：0.3%
最低结算金额：100
结算周期：实时结算
自动结算：开启
代付费率：0
转账费率：0
结算风控：开启
状态：启用
```

**商户费率计算示例**：

```
商户A（标准商户组，结算费率0.5%）：
- 收款1000元
- 平台收取：1000 × 0.5% = 5元
- 商户实收：1000 - 5 = 995元

商户B（VIP商户组，结算费率0.3%）：
- 收款1000元
- 平台收取：1000 × 0.3% = 3元
- 商户实收：1000 - 3 = 997元
```

### 2.5.5 配置案例

```
步骤1：查看申请信息
- 商户ID：10001
- 结算方式：支付宝
- 收款账号：13812345678

- 点击「审核通过并打款」

- 状态变为「已完成」

结果：金额返还商户账户，状态变为「已拒绝」

## 2.7 通道管理

**功能**：配置支付通道（最重要！）

### 2.7.1 概念解释

**插件 vs 通道**：

```
插件：是支付能力的代码实现（如支付宝官方接口）
通道：是插件的具体配置实例

例如：
插件「支付宝官方」
  ├── 通道「支付宝-费率0.5%-PC支付」
  ├── 通道「支付宝-费率0.6%-手机支付」
  └── 通道「支付宝-新客户专用-0.4%」
```

一个插件可以创建多个通道，每个通道可以设置不同的费率和限额。

### 2.7.2 通道列表

页面路径：管理后台 → 通道管理

```
┌──────────────────────────────────────────────────────────────────────┐
│ 通道管理                                                               │
├──────────────────────────────────────────────────────────────────────┤
│ [添加通道]                                                             │
├──────────────────────────────────────────────────────────────────────┤
│ ID │ 通道名称    │ 插件    │ 模式  │ 费率 │ 限额(元)    │ 状态 │ 操作   │
│ 1  │ 支付宝PC    │ alipay  │ 收款  │ 0.5% │ 1-10000    │ 启用 │[编辑]  │
│ 2  │ 支付宝H5    │ alipay  │ 收款  │ 0.6% │ 1-5000     │ 启用 │[编辑]  │
│ 3  │ 微信PC      │ wechat  │ 收款  │ 0.6% │ 1-10000    │ 启用 │[编辑]  │
└──────────────────────────────────────────────────────────────────────┘
```

### 2.7.3 添加/编辑通道

| 字段 | 说明 | 必填 | 示例 |
|------|------|------|------|
| 通道名称 | 标识名称 | 是 | 支付宝PC |
| 插件 | 支付插件 | 是 | alipay |
| 通道模式 | 收款/转账 | 是 | 收款 |
| 设备类型 | PC/H5/APP/全端 | 是 | PC |
| 分成比例(%) | 商户收款时费率 | 是 | 0.5 |
| 成本费率(%) | 平台付给上游成本 | 是 | 0.4 |
| 状态 | 启用/禁用 | 是 | 启用 |
| 日限额 | 每天最高收款 | 否 | 50000 |
| 单笔限额(最低) | 单笔最低 | 否 | 1 |
| 单笔限额(最高) | 单笔最高 | 否 | 10000 |

### 2.7.4 费率说明

| 费率类型 | 作用 | 示例（费率0.5%） |
|----------|------|-----------------|
| 分成比例 | 商户收款时扣除 | 收100元，商户实收99.5元 |
| 成本费率 | 平台付给上游 | 平台成本0.4元 |

### 2.7.5 通道模式

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| 平台代收 | 买家付款→平台账户→结算给商户 | 一般收款 |
| 商户直清 | 买家付款→直接到商户账户 | 商户自己有账户 |

### 2.7.6 配置案例

**案例1：创建支付宝PC支付通道**

```
通道名称：支付宝PC
插件：alipay
通道模式：平台代收
设备类型：PC
分成比例：0.5
成本费率：0.4
状态：启用
日限额：50000
单笔限额(最低)：1
单笔限额(最高)：10000
```

```
买家付款：100元
平台收取：100 × 0.5% = 0.5元
平台成本：100 × 0.4% = 0.4元
平台利润：0.5 - 0.4 = 0.1元
商户实收：100 - 0.5 = 99.5元
```

**案例2：创建微信H5支付通道**

```
通道名称：微信H5
插件：wechat
通道模式：平台代收

分成比例：0.6
成本费率：0.5
状态：启用
日限额：30000

# 第四章：费率计算

## 4.1 交易费率计算

### 公式

```
商户实收金额 = 订单金额 × (1 - 分成比例%)

例如：
订单金额 = 100元
分成比例 = 0.5%

商户实收 = 100 × (1 - 0.5%) = 100 × 0.995 = 99.5元
平台收取 = 100 × 0.5% = 0.5元
```

### 通道费率 vs 用户组费率

```
通道费率：商户使用某个通道收款时的费率
用户组费率：商户所属用户组的费率

实际计算时：
- 如果通道设置了分成比例，使用通道费率
- 如果用户组设置了结算费率，使用用户组费率
- 两者取其一，不叠加
```

## 4.2 结算费率计算

### 公式

```
实际到账金额 = 申请结算金额 × (1 - 结算费率%)

例如：
申请结算 = 1000元
结算费率 = 0.5%

手续费 = 1000 × 0.5% = 5元
实际到账 = 1000 - 5 = 995元
```

## 4.3 商户模式对费率的影响

### 模式说明

| 模式 | 值 | 计算公式 | 说明 |
|------|---|----------|------|
| 默认 | 0 | `费率 = 分成比例` | 标准费率 |
| 加费 | 1 | `费率 = 分成比例 + (100-分成比例)×50%` | 高风险加费 |
| 减费 | 2 | `费率 = 分成比例 × 50%` | 优质商户优惠 |

### 示例

假设分成比例为0.5%：

| 模式 | 计算 | 实际费率 |
|------|------|---------|
| 默认 | 0.5% | 0.5% |
| 加费 | 0.5% + (100-0.5%)×50% | 50.25% |

## 5.5 签名算法

### 签名格式

```
参数按字典序排列，用 & 连接，最后加上密钥

例如：
uid=10001&trade_no=order123&money=100.00&key=your_api_key

然后对上述字符串进行 MD5 加密
```

### 签名示例（Python）

```python
import hashlib

def make_sign(params, api_key):
    """生成签名"""
    # 1. 排序参数
    keys = sorted(params.keys())
    # 2. 拼接字符串
    string = '&'.join([f'{k}={params[k]}' for k in keys])
    # 3. 拼接密钥
    string += f'&key={api_key}'
    # 4. MD5加密
    sign = hashlib.md5(string.encode()).hexdigest()
    return sign
```

### 签名示例（JavaScript）

```javascript
function makeSign(params, apiKey) {
    // 1. 排序参数
    const keys = Object.keys(params).sort();
    // 2. 拼接字符串
    const string = keys.map(k => `${k}=${params[k]}`).join('&') + `&key=${apiKey}`;
    // 3. MD5加密
    return md5(string);
}
```
