## 系统简介

PayGo 是一个聚合支付网关平台，支持支付宝、微信支付等多种支付渠道接入。

### 核心功能

| 功能 | 说明 |
|------|------|
| 支付接入 | 一站式接入支付宝、微信等支付渠道 |
| 商户管理 | 多商户入驻，独立运营 |
| 订单管理 | 实时交易流水查询 |
| 结算管理 | 自动结算，灵活提现 |
| 通道管理 | 多通道轮询，智能路由 |
| 代付功能 | 商户资金自动/手动代付 |

### 商户管理

#### 商户列表

| 字段 | 说明 |
|------|------|
| UID | 商户唯一标识 |
| 商户名称 | 显示名称 |
| 用户组 | 所属分组（决定费率） |
| 余额 | 账户余额 |
| 费率 | 结算费率 |
| 支付权限 | 是否允许收款 |
| 结算权限 | 是否允许提现 |
| 状态 | 正常/禁用/待审核 |

#### 商户操作

**添加商户**：
1. 点击「添加商户」
2. 填写基本信息（用户名、邮箱、手机）
3. 设置登录密码
4. 选择用户组（决定默认费率）
5. 设置结算账户信息
6. 点击保存

**编辑商户**：
1. 点击商户「编辑」按钮
2. 修改相关信息
3. 点击保存

**商户字段说明**：

| 字段 | 说明 |
|------|------|
| 商户ID (UID) | 系统分配的唯一ID |
| API密钥 (Key) | 用于API签名认证 |
| 用户组 | 影响结算费率和通道权限 |
| 结算周期 | 按组内配置 |
| 支付权限 | 关闭后商户无法收款 |
| 结算权限 | 关闭后商户无法提现 |
| 代付权限 | 关闭后商户无法代付 |
| 状态 | 禁用后商户无法登录和交易 |

### 通道管理

**通道** = 具体支付渠道配置

#### 添加通道

1. 点击「添加通道」
2. 选择支付类型：
   - `1` = 支付宝
   - `2` = 微信支付
3. 选择绑定的插件（alipay / wxpay）
4. 填写通道名称，如「支付宝官方」
5. 设置商户费率（如 0.6%）
6. 配置通道参数（AppID、密钥等）
7. 设置限额：
   - 单笔最小限额
   - 单笔最高限额
   - 每日总限额
8. 保存

#### 通道字段说明

| 字段 | 说明 |
|------|------|
| 通道名称 | 自定义名称，如"支付宝当面付" |
| 插件 | 绑定的支付插件实现 |
| 支付类型 | 1=支付宝, 2=微信 |
| 商户费率 | 向商户收取的手续费 % |
| 成本费率 | 平台支付给通道的成本 % |
| 单笔限额 | 最小/最大单笔金额 |
| 日限额 | 当日累计上限 |
| 状态 | 启用/禁用 |

#### 通道优先级

- 系统根据「轮询配置」自动选择通道
- 也可以在订单中指定通道ID

## 开发人员手册

### 插件开发

#### 插件接口

```go
// plugin/interface.go
type Plugin interface {
    // 获取插件信息
    GetInfo() PluginInfo

    // 支付提交（PC端）
    Submit(params map[string]interface{}) (SubmitResult, error)

    // 移动端支付
    Mapi(params map[string]interface{}) (SubmitResult, error)

    // 异步回调
    Notify(tradeNo string, c *gin.Context) (NotifyResult, error)

    // 同步回调
    Return(tradeNo string, c *gin.Context) (ReturnResult, error)

    // 支付成功页面
    OK(tradeNo string) (string, error)

    // 退款
    Refund(params map[string]interface{}) (RefundResult, error)

    // 转账
    Transfer(params map[string]interface{}) (TransferResult, error)

    // 转账查询
    TransferQuery(params map[string]interface{}) (TransferQueryResult, error)
}
```

#### SubmitResult 返回类型

```go
type SubmitResult struct {
    Type string      // 返回类型
    URL  string      // 跳转URL或二维码内容
    Page string      // 页面模板
    Data interface{} // 扩展数据
    Msg  string      // 错误信息
}
```

Type 类型说明：
| Type | 说明 |
|------|------|
| `jump` | 重定向到URL |
| `qrcode` | 返回二维码内容 |
| `html` | 返回HTML表单 |
| `error` | 错误 |
| `page` | 跳转页面 |
| `scheme` | 跳转Scheme |
| `jsapi` | JSAPI调起参数 |
| `app` | APP调起参数 |
| `scan` | 扫码支付 |

#### 开发新插件

1. 创建插件目录：`plugin/channels/mypay/`
2. 实现插件接口：
```go
package mypay

import "paygo/src/plugin"

type MyPayPlugin struct{}

func init() {
    plugin.Register("mypay", func() plugin.Plugin {
        return &MyPayPlugin{}
    })
}

func (p *MyPayPlugin) GetInfo() plugin.PluginInfo {
    return plugin.PluginInfo{
        Name:     "mypay",
        Showname: "我的支付",
        Author:   "Developer",
        Types:    []string{"scan", "jsapi"},
    }
}

// 实现其他方法...
```

3. 在 `plugins.go` 中导入：
```go
import _ "paygo/src/plugin/channels/mypay"
```

## API 接口文档

### 商户支付接口

#### 创建订单

```
POST /api/pay/create
```

**请求参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| uid | int | 是 | 商户ID |
| key | string | 是 | API密钥 |
| trade_no | string | 是 | 商户订单号 |
| type | int | 是 | 支付类型 (1=支付宝, 2=微信) |
| money | float | 是 | 金额 |
| notify_url | string | 是 | 回调地址 |
| return_url | string | 否 | 返回地址 |
| name | string | 否 | 商品名称 |

**响应**：

```json
{
  "code": 0,
  "msg": "success",
  "trade_no": "202403011234567890",
  "pay_type": "alipay",
  "pay_info": "https://qr.alipay.com/xxx"
}
```

#### 订单查询

```
GET /api/pay/query?trade_no=xxx&uid=xxx&key=xxx
```

**响应**：

```json
{
  "code": 0,
  "trade_no": "202403011234567890",
  "status": 1,
  "money": 100.00,
  "pay_time": "2024-03-01 12:00:00"
}
```

#### 支付回调

```
POST /api/pay/notify/:trade_no
```

第三方支付回调，PayGo 通知商户服务器。

**商户服务器响应**：
- 成功：`return "success"`

## 常见问题

### 1. 商户如何对接支付？

1. 在商户后台获取 UID 和 API密钥
2. 调用 `POST /api/pay/create` 创建订单
3. 根据返回的 `pay_info` 调起支付
4. 支付成功后，系统 POST 回调到您的 `notify_url`

### 2. 如何添加新的支付渠道？

1. 开发插件（实现 Plugin 接口）
2. 在 `plugins.go` 中导入
3. 在管理后台添加通道，选择新插件
4. 配置通道参数

### 3. 商户费率如何设置？

- 在「用户组」中设置默认结算费率
- 编辑商户时可单独调整
- 结算时按商户实际费率计算

### 4. 回调地址是什么？

- 商户服务器接收支付通知的 URL
- 创建订单时通过 `notify_url` 参数传入
- 支付成功后，系统异步通知该地址

### 5. 如何处理售后/退款？

- 商户可在管理后台提交退款
- 或调用 `POST /api/pay/refund` API
- 退款后订单状态变为「已退款」

### 6. 系统支持哪些支付类型？

| type | 说明 |
|------|------|
| 1 | 支付宝 |
| 2 | 微信支付 |
| 3 | QQ钱包 |
| 4 | 银行卡 |

### 7. 如何配置测试支付？

在「系统设置」→「支付设置」中开启测试支付开关。
